## Princípios Obrigatórios

Sempre que trabalhar no código, siga estes princípios:

1. Código limpo, claro e legível.
2. Funções pequenas e com responsabilidade única.
3. Nomes descritivos para variáveis, funções, classes e arquivos.
4. HTML semântico e acessível.
5. CSS organizado, reutilizável e sem duplicações desnecessárias.
6. JavaScript modular, previsível e fácil de testar.
7. Evite soluções improvisadas ou frágeis.
8. Preserve funcionalidades existentes, salvo quando solicitado o contrário.
9. Não quebre o layout nem fluxos já existentes.
10. Antes de alterar, compreenda o contexto do projeto.

3. Explique brevemente o que será feito.
4. Faça a alteração necessária no código.
5. Revise se há efeitos colaterais.
6. Informe claramente o que foi alterado.

## Regras de Desenvolvimento

- Não crie código desnecessário.
- Não duplique lógica existente.
- Não remova funcionalidades sem necessidade.
- Não altere nomes de tabelas, rotas, variáveis globais ou APIs sem verificar impacto.
- Não use bibliotecas externas sem necessidade real.
- Prefira soluções simples, robustas e sustentáveis.
- Mantenha consistência visual e estrutural com o projeto existente.
- Corrija também pequenos problemas relacionados quando forem claramente parte da mesma causa.

## Padrões de Código

### JavaScript

- Use `const` e `let`, nunca `var`.
- Prefira funções puras quando possível.
- Evite funções longas.
- Separe responsabilidades.
- Trate erros adequadamente.
- Use async/await quando fizer sentido.
- Evite efeitos colaterais ocultos.
- Não deixe `console.log` desnecessário em produção.

### HTML

- Use tags semânticas.
- Mantenha boa hierarquia de títulos.
- Use atributos `aria` quando necessário.
- Garanta estrutura acessível para leitores de tela.
- Evite excesso de divs sem função.

### CSS

- Organize por responsabilidade.
- Evite estilos duplicados.
- Use variáveis CSS quando fizer sentido.
- Priorize responsividade.
- Garanta boa legibilidade visual.
- Evite `!important`, salvo em casos extremamente necessários.

Seu objetivo é entregar código profissional, estável, limpo, seguro, performático e fácil de manter, como faria um Engenheiro de Software Sênior responsável por um sistema real em produção.
